# 角色体系

本章节介绍 ROCK 系统的角色架构与配置方法。

## 角色架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户层                                    │
│         SDK 用户 / CLI 操作员 / 租户用户                         │
└─────────────────────────────┬───────────────────────────────────┘
                              │
┌─────────────────────────────▼───────────────────────────────────┐
│                      服务层                                      │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐           │
│  │ Admin 角色  │   │ Proxy 角色  │   │  Envhub     │           │
│  │  (管理)     │   │  (代理)     │   │  (注册中心) │           │
│  └─────────────┘   └─────────────┘   └─────────────┘           │
└─────────────────────────────┬───────────────────────────────────┘
                              │
┌─────────────────────────────▼───────────────────────────────────┐
│                      执行层                                      │
│         Worker 节点 / Rocklet 运行时代理                         │
└─────────────────────────────────────────────────────────────────┘
```

## 服务角色

Admin 服务支持两种运行角色，通过 `--role` 参数指定：

| 角色 | 用途 | 职责 |
|------|------|------|
| `admin` | 管理节点 | 沙箱创建/销毁、调度管理、资源分配、Ray 集群初始化 |
| `proxy` | 代理节点 | 命令执行、文件读写、会话管理、状态查询、端口转发 |

### 功能分离

| 操作类型 | Admin 角色 | Proxy 角色 |
|----------|------------|------------|
| 沙箱创建/销毁 | ✓ | ✗ |
| 调度任务 | ✓ | ✗ |
| 命令执行 | ✓ | ✓ |
| 文件操作 | ✓ | ✓ |
| 状态查询 | ✓ | ✓ |
| 端口转发 | ✓ | ✓ |
| 会话管理 | ✓ | ✓ |

### 部署建议

- 生产环境建议部署多个 Proxy 节点分担读请求
- Admin 节点通常部署 1-2 个即可

## 组件角色

| 组件 | 角色 | 说明 |
|------|------|------|
| **SDK** | 客户端 | 开发者使用的环境开发工具包 |
| **CLI** | 操作员 | 命令行管理工具 |
| **Admin** | 编排器 | 沙箱生命周期调度节点 |
| **Worker** | 执行器 | 物理资源分配节点 |
| **Rocklet** | 运行时代理 | 容器内轻量级代理服务 |
| **Envhub** | 注册中心 | 环境数据存储与 CRUD 服务 |

## 租户用户标识

通过 HTTP 请求头传递租户信息：

| Header | 默认值 | 说明 |
|--------|--------|------|
| `X-User-Id` | `default` | 用户标识 |
| `X-Experiment-Id` | `default` | 实验项目标识 |
| `X-Namespace` | `default` | 命名空间（隔离） |
| `X-Key` | `default` | 授权令牌（加密） |
| `X-Cluster` | `default` | 集群标识 |

### 多租户隔离

- 相同 `namespace` 下的资源相互隔离
- `X-Key` 用于权限验证

## 环境变量配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ROCK_ADMIN_ENV` | `dev` | 运行环境：local / dev / test |
| `ROCK_ADMIN_ROLE` | `write` | Admin 角色标识（用于监控） |
| `ROCK_WORKER_ENV_TYPE` | `local` | Worker 类型：local / docker |
| `ROCK_WORKER_ROCKLET_PORT` | 无 | 自定义 Rocklet 端口 |

## 典型部署架构

### 单节点部署

```
Admin (role=admin) + Rocklet
```

### 高可用部署

```
                    ┌─────────────┐
                    │ 负载均衡器  │
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │   Admin     │ │   Proxy     │ │   Proxy     │
    │ (role=admin)│ │ (role=proxy)│ │ (role=proxy)│
    └─────────────┘ └─────────────┘ └─────────────┘
           │               │               │
           └───────────────┼───────────────┘
                           ▼
                    ┌─────────────┐
                    │    Redis    │
                    │ (共享状态)  │
                    └─────────────┘
```

## 授权机制

ROCK 采用令牌授权机制，而非传统 RBAC：

- **授权验证**：通过加密令牌 `X-Key` 验证
- **命名空间隔离**：通过 `X-Namespace` 实现多租户隔离
- **读写分离**：Admin/Proxy 角色实现管理/运行操作分离

## 监控标识

所有监控指标包含角色属性，便于区分：

| 属性 | 说明 |
|------|------|
| `host` | 主机名 |
| `pod` | Pod 标识 |
| `ip` | IP 地址 |
| `env` | 环境标识 |
| `role` | 角色标识 |
