# 端口配置

本章节介绍 ROCK 系统的端口架构与配置方法。

## 端口架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                        客户端层                                  │
│              SDK / CLI / HTTP/WebSocket 客户端                   │
└─────────────────────────────┬───────────────────────────────────┘
                              │
┌─────────────────────────────▼───────────────────────────────────┐
│                       服务层                                     │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐           │
│  │   Admin     │   │  Rocklet    │   │   Envhub    │           │
│  │   :8080     │   │  :22555     │   │   :8081     │           │
│  └─────────────┘   └─────────────┘   └─────────────┘           │
└─────────────────────────────┬───────────────────────────────────┘
                              │
┌─────────────────────────────▼───────────────────────────────────┐
│                     容器层                                      │
│         动态端口映射 (find_free_port)                           │
└─────────────────────────────────────────────────────────────────┘
```

## 服务端口

### 默认端口分配

| 服务 | 默认端口 | 说明 |
|------|----------|------|
| Admin | 8080 | 主管理服务 |
| Rocklet | 22555 | 沙箱运行时代理 |
| Rocklet WebSocket | 8000 | 容器内 WebSocket 服务 |
| Envhub | 8081 | 环境注册中心服务 |

### 端口常量

| 端口类型 | 端口号 | 用途 |
|----------|--------|------|
| PROXY | 22555 | Rocklet 代理通信 |
| SERVER | 8080 | WebSocket 服务 |
| SSH | 22 | SSH 服务（容器内部） |

## 配置方式

### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ROCK_BASE_URL` | `http://localhost:8080` | Admin 服务 URL |
| `ROCK_ENVHUB_BASE_URL` | `http://localhost:8081` | Envhub 服务 URL |
| `ROCK_WORKER_ROCKLET_PORT` | 无 | 自定义 Rocklet 端口（未设置时使用 22555） |

### 命令行参数

**Admin 服务：**

```bash
python -m rock.admin.main --port 8080 --env dev --role admin
```

**Rocklet 服务：**

```bash
rocklet --host 0.0.0.0 --port 8000
```

**Envhub 服务：**

```bash
python -m rock.envhub.server --host 0.0.0.0 --port 8081
```

### 配置文件（K8s）

```yaml
k8s:
  ports:
    proxy: 8000    # HTTP API 端口
    server: 8080   # WebSocket 服务端口
    ssh: 22        # SSH 服务端口
```

## 端口转发

### API 端点

| 端点 | 用途 |
|------|------|
| `/sandboxes/{id}/portforward?port={port}` | 外部访问端点 |
| `/portforward?port={port}` | Rocklet 内部端点 |

### 端口转发规则

| 规则 | 值 | 说明 |
|------|-----|------|
| 最小端口 | 1024 | 排除特权端口 (0-1023) |
| 最大端口 | 65535 | 端口上限 |
| 排除端口 | 22 | SSH 端口禁止转发 |

### 转发流程

```
客户端 WebSocket → Admin Proxy → Rocklet /portforward → 容器内 TCP 端口
```

## 动态端口分配

部署容器时，ROCK 自动分配可用端口：

- 使用 `find_free_port()` 查找可用端口
- 维护已分配端口注册表避免冲突
- 将容器内部端口动态映射到宿主机端口

### 容器端口映射

| 容器内部端口 | 宿主机映射 | 说明 |
|-------------|-----------|------|
| 22555 (PROXY) | 动态分配 | Rocklet 代理端口 |
| 8080 (SERVER) | 动态分配 | WebSocket 服务 |
| 22 (SSH) | 动态分配 | SSH 服务 |

## 端口冲突处理

### 验证规则

1. 端口必须 `>=` 1024（非特权端口）
2. 端口必须 `<=` 65535
3. 端口不能在排除列表中（SSH: 22）

### 冲突避免

- 动态端口分配使用系统分配的可用端口
- 全局端口注册表跟踪所有已分配端口
- 支持可配置的重试机制

### 高可用配置

```
                    ┌─────────────┐

           ┌───────────────┼───────────────┐

    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐

    │   :8080     │ │   :8080     │ │   :8080     │
    └─────────────┘ └─────────────┘ └─────────────┘
```

### 端口规划建议

| 环境 | Admin | Proxy | Rocklet | Envhub |
|------|-------|-------|---------|--------|
| 开发环境 | 8080 | - | 22555 | 8081 |
| 生产单节点 | 8080 | - | 22555 | 8081 |
| 生产高可用 | 8080 | 8080 | 22555 | 8081 |

## 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 端口已被占用 | 服务冲突 | 更换端口或停止冲突服务 |
| 连接被拒绝 | 服务未启动 | 检查服务状态 |
| 端口转发失败 | 端口不在允许范围 | 使用 `>=` 1024 且 `!=` 22 的端口 |

### 端口状态检查

```bash
# 检查端口占用
netstat -tlnp | grep -E '8080|22555|8081'

# 检查服务状态
curl http://localhost:8080/health
```
